# IT Project Planner — 4-Stage Planning Suite

A structured skill suite that guides Claude through the full lifecycle of IT project planning — from initial requirements gathering to sprint-ready architecture and Jira task breakdown.

## Skills

| Skill | Stage | What it produces |
|-------|-------|-----------------|
| `/project-init` | Stage 1 | Requirements interview → 3 project variants → confirmed tech stack |
| `/project-research` | Stage 2 | Competitor research, UX personas, hypotheses, value proposition, 25–36 user stories |
| `/project-interviews` | Optional | Simulated UX interviews for each persona — surfaces pain points and contradictions |
| `/project-mvp` | Stage 3 | MVP matrix with Impact/Complexity/Stack Risk scoring, release stages, KPIs |
| `/project-sprints` | Stage 4 | Jira board, epics, sprint plan with typed tasks, system architecture with Mermaid diagrams |

## Usage

Run stages in order — each skill validates that the previous stage output is present.

```
/project-init       → define the project, pick a variant
/project-research   → competitor research, personas, user stories
/project-interviews → (optional) simulate persona interviews
/project-mvp        → prioritize stories, plan releases, set KPIs
/project-sprints    → sprint breakdown, task cards, architecture
```

Each skill accepts the project name as an argument:

```
/project-init my-app
/project-research my-app
/project-mvp my-app
/project-sprints my-app
```

## Output documents

| Document | Generated by |
|----------|-------------|
| `[ProjectName]-requirements.md` | `/project-init` |
| `[ProjectName]-research.md` | `/project-research` |
| `[ProjectName]-interview-guide.md` | `/project-research` |
| `[ProjectName]-interviews.md` | `/project-interviews` |
| `[ProjectName]-mvp.md` | `/project-mvp` |
| `[ProjectName]-sprints.md` | `/project-sprints` |
| `[ProjectName]-architecture.md` | `/project-sprints` |

## Features

- **Language-aware** — responds in the user's language; all documents generated in English
- **Stack-aware** — calibrates complexity, velocity, and capacity to team size and tech stack
- **AI-assistance mode** — `/project-init` expands the stack and marks AI-assisted components when opted in
- **Educational vs standalone** — separate paths for learning projects vs. production builds
- **Mermaid diagrams** — architecture output renders in GitHub, Confluence, and Notion
- **Jira-compatible** — task IDs follow `[PROJ]-N` format; all task types included (Story/Feature/Spike/Tech Debt/Research/Bug)
